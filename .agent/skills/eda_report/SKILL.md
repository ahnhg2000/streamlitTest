# EDA 시각화 및 인사이트 도출 스킬 (EDA Visualization & Insight Rule)

이 스킬은 데이터 분석 보고서의 시각적 품질을 극대화하고, 데이터의 핵심 피처(Feature)를 발굴하여 비즈니스적/의학적 시사점을 도출하는 공통 규칙을 정의합니다.

## 1. 시각화 및 파일 저장 규칙 (Visualization & Storage Rules)
모든 분석 보고서는 텍스트뿐만 아니라 **시각화 이미지**를 반드시 포함해야 합니다.
- **이미지 자동 저장**: 대시보드에서 생성된 주요 그래프(`Heatmap`, `DistPlot`, `Scatter`)는 `report/images/` 폴더에 `{파일명}_{타입}.png` 형식으로 자동 저장됩니다.
- **이미지 임베딩**: 저장된 이미지는 Markdown 보고서 내에 `![설명](path)` 문법을 통해 시각적으로 배치됩니다.
- **시각화 선택 기준**:
  - 상관관계: 전체 변수 간의 흐름을 파악하는 인터랙티브 히트맵.
  - 중요도: 타겟(Target) 변수에 가장 큰 영향을 미치는 'Top 3 피처' 바 차트.

## 2. 핵심 피처 발굴 규칙 (Feature Importance Rules)
데이터 내에서 가장 중요한 컬럼을 선정하는 기준을 수립합니다.
- **상관계수 필터링**: 타겟 변수와 절대적 상관도(Abs Correlation)가 높은 순으로 추출.
- **다변량 교차 분석**: 단일 변수의 영향뿐만 아니라, 두 변수가 결합했을 때의 시너지 효과(Interaction Effect) 분석.
- **피처 의미 부여**: 기술 통계량의 차이(예: 양성/음성 그룹 간 평균 차이)가 뚜렷한 변수를 '핵심 동인(Key Driver)'으로 선정.

## 3. 인사이트 및 시사점 구조화 (Insight Structure)
분석 내용 정리 시 다음 3단계 구조를 따릅니다.
1. **발견(Findings)**: "혈당 수치가 140 이상인 그룹에서 당뇨 양성 판정 비율이 3배 높게 나타남." (객관적 사실)
2. **의미 및 시사점(Significance)**: "이는 혈당이 당뇨 예방 및 진단의 결정적 임계치임을 시사함." (데이터 해석)
3. **제안(Actionable Insight)**: "고위험군 선별을 위해 BMI 30 이상인 대상자의 정기 혈당 모니터링 체계가 필요함." (실행 방안)

## 4. 보고서 품질 체크리스트 (Quality Checklist)
- [ ] 중요한 피처 TOP 3가 명확히 명시되었는가?
- [ ] 히트맵을 통해 전체적인 변수 간 관계를 설명했는가?
- [ ] 시각화 이미지가 보고서 문맥에 맞게 포함되었는가?
- [ ] 전문 용어와 데이터 기반의 상세 설명이 결합되었는가?
